# Frank's Adventure Design Language

> 本文汇总当前项目所有已出现设计语言的共性，作为后续 UI、卡牌、事件、董事会、动效与素材扩展的统一参考。具体规则仍以 `GAME_DESIGN_FOUNDATION.md`、`BATTLEFIELD_DESIGN.md`、`BOARD_MEETING_DESIGN.md`、`CARDS_DESIGN.md` 与代码实装为准。

## 1. 核心气质

**复古像素 × 现代 Web 特效 × 公司经营荒诞感**。

界面不走传统 SaaS 的干净扁平路线，而是把“办公室经营”“融资董事会”“卡牌爆分”做成带街机感的商业冒险。它应该同时有三种味道：

- **像素游戏感**：硬边、斜切角、扫描线、像素字体、像素头像、像素图标。
- **商业经营感**：现金、估值、AP、部门、董事会、商业模式、阶段融资这些信息要清晰、可比较、可决策。
- **爆分爽感**：平时信息克制，结算、抽卡、升阶、稀有卡出现时允许高亮、抖动、扫光、粒子、弹字和 2.5D 抬升。

一句话：**像一台被投资人塞进街机厅里的 CEO 模拟器。**

## 2. 信息层级

主界面遵循“极简可读，细节藏入 hover”的原则。

- 第一眼只露出当前决策必须知道的核心状态：日期、估值/阶段进度、现金、AP。
- 细节拆解通过浮动 tooltip、抽屉、弹窗、卡牌 hover、图鉴等二级界面承载。
- 面板不应堆满解释文字；玩家需要的是“现在能做什么、代价是什么、结果可能怎样”。
- 战斗界面强调月度循环与产线布置；董事会界面强调资本分配、牌库整理与长期战略。

## 3. 视觉骨架

所有主要 UI 都共享同一套像素硬边结构：

- **斜切角**：卡牌、按钮、财务部、BM 卡、弹窗、教程卡统一使用小尺寸 `clip-path` 斜切角。
- **厚像素边框**：常见 2-4px 边线，边角用小三角补齐，形成“游戏 UI 边框”。
- **无圆角或极少圆角**：卡牌、按钮、游戏面板默认不用圆角；只有工具型小按钮或现代浮层可少量使用 8px 以内圆角。
- **硬阴影**：优先用 `drop-shadow` 或 2-14px 的实心偏移阴影，表达像素卡片的厚度。
- **透明深色底**：游戏浮层多用深蓝黑、半透明、轻微渐变，而不是纯白面板。

## 4. 空间与透视

当前语言不是纯 2D 平铺，而是轻 2.5D 舞台。

- 主战斗场景以办公室/会议室照片或像素化背景为底，上面叠加 CRT、扫描线、暗角和地面网格。
- 产线、卡牌、按钮、HUD 数字都有轻微 `rotateX/rotateY/translateZ`，形成桌面上的实体感。
- 手牌是扇形/层叠牌组，hover 或选中时上浮、放大、正面朝向玩家。
- 产线 A/B 通过上下层级、缩放、透明度和 Z 轴区分“生产中”和“可操作”。
- 重要反馈从画面中心或卡牌本体爆出，而不是只在文本日志里更新。

## 5. 色彩共性

底色以深色办公室和夜间经营压力为主，高饱和色只用来表达状态和奖励。

基础色：

| 用途 | 色彩倾向 |
|---|---|
| 背景 | 深夜蓝、墨青、深蓝黑 |
| 主文字 | 近白、淡灰蓝 |
| 次级文字 | 灰蓝 |
| 正向/现金/利润 | 绿色 |
| 警示/Burn/风险 | 红色 |
| AP/启动/行动 | 橙色 |
| 信息/协同 | 蓝色 |
| 奖励/传奇/收割 | 金色 |
| 管理/品牌/HUD 高亮 | 紫色 |

稀有度是全局语义，不随界面变化：

| 稀有度 | 颜色 |
|---|---|
| common | 灰 |
| rare | 蓝 |
| elite | 紫 |
| epic | 橙 |
| legendary | 金 |

部门与卡牌类型也有固定倾向：

- 研发：偏紫/深红科技感。
- 销售：暖橙/珊瑚/肉桂色。
- 运营：棕绿/土色/组织管理感。
- 服务：靛蓝/专业服务感。
- 功能：浅色或琥珀，偏工具牌。

## 6. 字体与文字

字体语言以像素字体为主，中文也要保持块状、游戏化、短句化。

- 主字体：`Fusion Pixel SC` / `Fusion Pixel`。
- 标题、Logo、菜单可使用更夸张的 `Adventure` 字体。
- 数字是界面的主角，现金、估值、AP、产出可以比说明文字大很多。
- 文案偏“公司经营吐槽”，但按钮与关键状态必须直白。
- 不在主界面写长教程；新手引导、tooltip、图鉴承载解释。

## 7. 卡牌语言

卡牌是视觉系统的中心物件，所有系统都应围绕“卡是实体资产”来设计。

- 卡牌有固定硬边、厚阴影、稀有度边框、部门底色、头像/图标、AP、产出/主效果。
- 卡牌 hover 时出现扫光、上浮、发亮、细节 tooltip。
- 精英及以上可以有更明显的光圈；传奇和创始人允许特殊跑马灯边框。
- 同名卡也可以因为数值 roll、词条、部门和稀有度呈现不同“具体的人”的感觉。
- 卡面信息优先级：名称/部门、稀有度、AP、头像、主产出或主效果、短效果。

## 8. 交互反馈

交互必须有“按得下去、拿得起来、放得进去”的触感。

- 可点击元素 hover：上浮 2-4px、变亮、边框或阴影增强。
- active/按下：下沉、阴影变短。
- 拖拽：透明度降低、亮度变化、卡牌保持实体感。
- 选中：金色或稀有度色强调，必要时出现外圈光效。
- 禁用：灰度、低透明、保留形状但降低欲望。
- 重大事件：允许中心弹字、镜头轻震、粒子、扫光、短暂音画高潮。

## 9. 动效节奏

动效服务于经营节奏，不做纯装饰堆叠。

- 菜单和 Logo 可以夸张入场，建立冒险感。
- 常规按钮动效短促，约 120-180ms。
- 卡牌抽取/入手可以更戏剧化，约 500-800ms。
- 月末结算是高潮，允许 3-5 秒链式触发：P1 到 P5、倍率、总产出、估值变化依次出现。
- 背景动效保持低强度：CRT 闪动、光带、轻微视差，不能抢过卡牌和数字。

## 10. 布局原则

界面应该像一张可操作的办公桌，而不是营销落地页。

- 主战斗：顶部 HUD，中间办公室产线，底部手牌，侧边放候选/日志/辅助状态。
- 董事会：顶部阶段与现金，核心财务/商店/HR/BM/事件分区，强调“会议桌上的运营决策”。
- 图鉴/列表：密度可以更高，但仍沿用像素卡片、稀有度色和短信息块。
- 移动端优先保留核心操作链：HUD → 产线 → 手牌 → 结算按钮。

## 11. 资产风格

素材需要服务“办公室经营 roguelike”的主题。

- 背景：办公室、会议室、董事会、商业场景，允许照片像素化或高质量像素绘制。
- 人物：像素头像或模块化 SVG 像素高管，服饰/配件体现部门和职级。
- 图标：优先使用清晰可识别的商务/资源/行动图标；像素图标要保持硬边。
- 卡牌插画：部门、稀有度、卡种应能一眼区分。
- 特效：金色、绿色、蓝色、紫色用于成功、利润、协同、稀有度，红色只给风险和告急。

## 12. 设计禁忌

- 不要把主界面做成传统数据看板，避免密密麻麻的 KPI 表格。
- 不要用大圆角、软糯玻璃卡片破坏像素硬边。
- 不要让所有区域都发光；发光必须表达稀有、可操作、告急或结算高潮。
- 不要用长段说明占据游戏画面；解释放进 tooltip、教程、图鉴。
- 不要引入与商务办公室无关的奇幻装饰，除非它是明确的卡牌/事件梗。
- 不要让颜色失去语义：红是风险，绿是收益，金是奖励，蓝是信息，橙是行动，紫是管理/稀有/品牌感。

## 13. 一句话检查

新增任何界面或素材前，先问：

**它是否像一个像素化商业冒险里的真实经营工具，并且在玩家做关键决策时只突出最该看的信息？**

如果答案是肯定的，通常就符合 Frank's Adventure 的设计语言。
