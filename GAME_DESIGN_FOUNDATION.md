## §A 核心机制（已锁定，不再变动）

### A.1 时间结构

```
1 局 = 1 个月（核心游戏循环单位）
1 关 = 6 局 = 6 个月（半年）
全游戏 = 10 关 = 60 局 = 5 年公司发展史
（v0.3 当前实装 3 关：A/B/C 轮融资；关 4-10 待补，详见 §B.2 数值平衡）
```

每关对应一个里程碑：

| 关卡 | 里程碑 | 主题 |
|---|---|---|
| 1 | A 轮融资 | 起步打 PMF |
| 2 | B 轮融资 | 规模化 |
| 3 | C 轮融资 | 抢市场 |
| 4 | D 轮融资 | 行业地位 |
| 5 | IPO | 上市冲刺 |
| 6 | 百亿市值 | 二次曲线 |
| 7 | 千亿市值 | 生态构建 |
| 8 | 世界 500 强 | 全球化 |
| 9 | 国际巨头 | 跨界征服 |
| 10 | 行业第一 | 终极对决 |

### A.2 产线机制

- **2 条产线槽位**，每条 **5 格单行**
- **强制轮转**：产线进入生产后玩家无法调整，必须布置另一条
- **生产时长**：2 个月（2 局），之后员工下线
- **冷却时长**：员工下线后冷却 1 个月，期间不可用
- **冷却结束**：员工回**牌堆**（不是手牌），下次抽牌可能抽到

**轮转示意**：

```
局 1: 布置产线 A → A 第 1 代生产
局 2: A 第 1 代仍生产中 → 强制布置产线 B → B 第 1 代生产
局 3: A 第 1 代下线进冷却 → 布置 A 第 2 代 → B 第 1 代仍生产
局 4: A 第 2 代生产 / B 第 1 代下线进冷却 / 布置 B 第 2 代
局 5: A 第 2 代下线 / 布置 A 第 3 代 / B 第 2 代生产
局 6: 关卡末局 / 布置 B 第 3 代 / 结算
```

**结算机制**：每月所有"生产中"产线全部触发结算（机制 X，非累积）

### A.3 三大资源 / 约束

| 名称 | 符号 | 性质 | 用途 | 跨局保留 |
|---|---|---|---|---|
| **现金** | 💴 ¥ | 局内核心资源 | 产线产出、维持费扣除、**关卡通关条件** | 关卡末结算后清零 |
| **管理力（AP）** | ⚡ | 局内每月约束 | 限制每月产线布置规模 | 每回合未使用 AP 的一半进入下回合（封顶 +5，对应 `GAME_CONFIG.carryApCap`）|
| **战略预算** | 💰 | 购卡/购物专用资源 | 局内招聘市场 + 关卡末商店（传奇卡 + 新商业模式）| 待定（商店系统设计时确定）|

**关键变化（v0.3）**：
- ❌ **彻底删除"估值 ⭐"系统**——简化资源轴
- ✅ ¥ 角色纯化为：**产线产出 → 累计 → 关卡通关**
- ✅ 战略预算 💰 角色纯化为：**购买行为专用货币**（局内招聘 + 关卡末商店）
- ✅ ¥ 和 💰 **职能完全分离**，互不重叠

**资源关系新版**：

```
产线激活 → 产出 ¥
            │
            ├→ 计入累计 ¥ 收入（通关指标）
            └→ 进入玩家 ¥ 余额（扣维持费）

战略预算 💰 → 局内：招聘市场三选一买新卡进牌堆
           └→ 关卡末商店：买传奇卡、新商业模式、永久升级

¥ 用途：维持费 + 关卡通关
💰 用途：所有"购买卡牌或升级"的行为
```

**职能分离的好处**：
- 玩家不会陷入"¥ 用来招人还是攒着通关"的两难
- ¥ 越多越接近通关，路径清晰
- 💰 是独立的"投资资金"，与日常运营脱钩
- 主题契合：¥ = 公司账上现金；💰 = 投资人给的弹药

### A.4 牌组系统

- **起始手牌**：7 张（固定起始牌组，对应 `STARTER_HAND` 展开）
- **每月抽牌**：3 张（从牌堆抽，对应 `GAME_CONFIG.drawPerMonth`）
- **手牌上限**：10 张（超出强行弃，玩家自选弃哪张）
- **起始牌堆**：15 张（含起始手牌外的所有起始卡）
- **员工状态流转**：
  ```
  [牌堆] → 抽 → [手牌] → 布置 → [产线生产中] 
                                    │ (2 个月后)
                                    ↓
                                [冷却 1 个月]
                                    │
                                    ↓
                                [回牌堆] → 可能再被抽到
  ```
- **服务卡/功能卡**：同样适用上述流转

### A.5 每月流程

```

═══════════════════════════════════════

  · 显示本月特殊规则

  · 刷新 3 张明牌候选
  · 每月最多签约 1 张，用 💰 战略预算购买 → 进入牌堆
═══════════════════════════════════════

  · 从牌堆抽 3 张进手牌
  · 超 10 张上限强行弃

  · 系统提示本月可布置哪条产线（强制约束）

  · AP 实时消耗，超 5（+保留）无法放置
  · 预览栏实时显示本月预估 ¥ 收入

```

### A.6 产线产出公式

```
月 ¥ 收入 = ∑(各位置基础产出 × 所有适用乘数)
累计 ¥ 收入 = 关卡内各月 ¥ 收入之和
```

**说明**：
- 产线直接产出 ¥，不再有"业绩分"或"⭐ 估值"中间层
- 累计 ¥ 收入是**单调累加**的指标，用于关卡通关判定
- 玩家手头实时 ¥ 余额 = 累计 ¥ 收入 - 累计支出（维持费等）
- 通关判定看的是**累计收入**而非余额

### A.7 产线计分机制

**位置加成**（5 格位置本身属性）：

| 位置 | 加成 |
|---|---|
| P1 启动位 | +20% 基础产出 |
| P2 普通 | 无 |
| P3 中枢位 | 给左右邻 +20% |
| P4 普通 | 无 |
| P5 收割位 | 若本卡贡献 ≥60% 总分，本卡 ×1.5 |

**邻接 buff**：
- 卡牌自身效果（如 → 右邻 +X%、← 左邻 +X%、↔ 双向 +X%）
- 同部门相邻协同（默认 +20%，可被卡效果覆盖）

**叠加规则**：
- 所有 buff 同时生效（静态计算，先算 buff 再激活）
- 同来源不重复
- 卡牌自身效果与位置加成可叠加
- 最终乘数 = ∏(所有适用加成)

### A.8 关卡通关条件

- **基础通关**：关卡末局**累计 ¥ 收入** ≥ 关卡目标 ¥
- **优秀通关**：累计 ¥ 收入 ≥ 关卡目标 × 1.5
- **卓越通关**：累计 ¥ 收入 ≥ 关卡目标 × 2.0

**说明**：
- "累计 ¥ 收入" = 关卡 6 局产线产出的 ¥ 总和（**不扣支出**）
- 玩家手头实时 ¥ 余额可能因维持费下降，但累计收入是单调累加的指标
- 不同评级影响**关卡末战略预算 💰 的发放**（评级越高，战略预算越多）
- 评级具体奖励详见 §B.4 商店系统（待设计）

### A.9 战略预算 💰

**战略预算的获取**：

- 关卡开始时一次性发放（基础预算 + 上关评级奖励）
- 公式：💰 基础 20 + 评级奖励（基础通关 0 / 优秀 +5 / 卓越 +10）
- 详细发放规则见 [BOARD_MEETING_DESIGN.md §2.2](./BOARD_MEETING_DESIGN.md#22-关末--发放公式)

**战略预算的使用**：

| 用途 | 时机 | 说明 |
|---|---|---|
| 局内招聘市场 | 每月 Phase 3 | 购买新卡进牌堆 |
| 董事会会议 · 投资部 | 关末 | epic / 传奇单卡 + 卡包 |
| 董事会会议 · 人事部 | 关末 | 升职 / 解雇 |
| 董事会会议 · 商学院 | 关末 | 购买商业模式（全局规则修改器）|

> 关间「董事会会议」环节完整设计见 [BOARD_MEETING_DESIGN.md](./BOARD_MEETING_DESIGN.md)

**跨关保留**：**未使用 💰 全部保留到下关**（无清零，无上限，类似 Balatro 金币机制）。配合 BOARD_MEETING_DESIGN 中的价格曲线 + 内容解锁曲线反向制衡，避免攒爆。

### A.10 ¥ 跨关处理

**关卡末**：玩家手头剩余 ¥ **清零**（不跨关）。

**理由**：
- ¥ 是局内运营资源，关卡间的"投资资金"是战略预算
- 跨关清零避免数值膨胀
- 主题契合：每个发展阶段是独立的"账期"

## §C 设计哲学（贯穿原则，不可违背）

1. **公司经营主题契合**：所有机制必须能用商业逻辑解释
2. **决策密度**：每月都要有有意义的决策，不能空闲
3. **决策深度**：相同手牌应有 ≥3 种合理打法
4. **爆分爽感**：每月结算有数字/视觉/音效高潮
5. **构筑成就感**：玩家能看到自己公司"长大"的过程
6. **roguelike 方差**：每局开始应有变量，10 局不相同
7. **失败温柔化**：失败不让玩家觉得"白玩"
8. **学习曲线平缓**：新机制每关只引入 1 个
