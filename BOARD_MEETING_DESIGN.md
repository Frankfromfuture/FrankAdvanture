## §1 概览

### 1.1 定位与设计目标

「董事会会议」是关与关之间的关间环节，对应商业叙事是「上一轮融资完成，董事会召开战略会议，决定下一阶段的资金投入方向」。

**设计目标**：

1. **Roguelike 节奏锚点**：每关后派发奖励 / 决策机会，避免 10 关连推疲劳
2. **跨关成长曲线**：通过升级 / 解雇 / 全局 buff，让牌库随关推进而进化
3. **叙事节拍器**：通过随机事件提供新鲜感与代入感
4. **资源消费出口**：让 💰 战略预算有充分消费场景，避免攒钱无意义

### 1.2 触发时机

```
关 N · Phase 7（月末结算） 完成
    │
    ├─ if N == 10 → 进入【全游戏通关结算】（不触发会议）
    │
    └─ else → 进入【董事会会议】
                │
                └─ 完成后 → 关 N+1 · Phase 1
```

整局游戏共 **9 次**「董事会会议」（关 1-2、2-3、…、9-10 之间各 1 次）。

### 1.3 Hub 结构

进入会议后玩家看到一个**单页 Hub**，包含 4 个站点：

```
┌──────────────────────────────────────────────────────────────┐
│ 💰 余额: 47    [进入下关 →]              第 3 轮 → 第 4 轮      │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐         │
│   │  投资部  │  │  人事部  │  │  商学院  │  │  董事访谈 │       │
│   │  Shop   │  │  HR     │  │  School │  │  Events  │       │
│   │         │  │         │  │         │  │   ⭐     │       │
│   └─────────┘  └─────────┘  └─────────┘  └─────────┘         │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

- **顶栏常驻**：💰 余额 + 「进入下关」按钮
- **入场强制事件**：进入 Hub 时先弹 1 个随机叙事事件 modal（董事访谈），玩家必选 1 项后关闭，事件结果以 toast 写顶栏
- **4 站点可自由进出**：点击展开抽屉式面板（不切页，💰 余额始终可见）
- **「进入下关」按钮**：始终可点击，无强制消费门槛；剩余 💰 **全部保留**到下关

### 1.4 设计原则

| 原则 | 表现 |
|---|---|
| **决策不疲劳** | 4 站点并列，可跳过；事件强制 1 次结束 |
| **横向比价** | Hub 单页布局，💰 余额顶栏可见，便于权衡 |
| **节奏正反馈** | 关末刚结算 → 立刻奖励派发 → 再决策下关投入 |
| **后期更值钱** | 商品价格随关递增，💰 全保留但购买力不无脑膨胀 |

## §2 经济流

### 2.1 💰 战略预算结转规则

- **未使用 💰 全部保留到下关**（无清零，无上限）
- 类比 Balatro 金币：玩家可以选择"前期节省、后期爆发"
- 配合本节 §2.3 价格曲线 + §7 内容解锁曲线反向制衡，防止"攒爆"

### 2.2 关末 💰 发放公式

| 来源 | 数值 |
|---|---|
| 基础预算 | 💰 20（每关固定，**不随关递增**） |
| 评级奖励 - 基础通关 | +💰 0 |
| 评级奖励 - 优秀通关（×1.5） | +💰 5 |
| 评级奖励 - 卓越通关（×2.0） | +💰 10 |

整局期望累计获取：9 关 × (20 + 平均 4) ≈ **💰 216**。

> **关末发放保持线性**：不引入指数增长曲线，避免后期通货膨胀。玩家想买更多 → 必须靠评级提升（高难度自我挑战），符合 roguelike 内驱激励。

### 2.3 商品价格曲线总览

| 商品类型 | 公式 | 关 1 价 | 关 9 价 |
|---|---|---|---|
| epic 单卡（投资部 A 槽） | 💰 = N + 8 | 9 | 17 |
| 传奇卡（投资部 B 槽） | 💰 = N × 4 + 15 | 19 | 51 |
| 卡包（投资部 C-E 槽） | 见 §3 | 4-10 | 4-10 |
| 商业模式（商学院） | 普通 N+6 / 稀有 N+12 / 传奇 N+18 | 7 / 13 / -- | 15 / 21 / 27 |
| 升职（人事部 ①） | 目标稀有度等级 × 5 | 10-20 | 10-20 |
| 升职（人事部 ②词缀） | 💰 8（固定） | 8 | 8 |
| 解雇（人事部） | 关 1-3: 3 / 关 4-6: 5 / 关 7+: 8 | 3 | 8 |
| 刷新（投资部 / 商学院） | 💰 5 / 💰 4（不封顶次数） | 5/4 | 5/4 |

详细分模块价格表见 §3-§6。

## §3 模块 1：投资部（Shop）

### 3.1 槽位结构（5 槽）

每次进入会议或刷新后，投资部展示：

```
┌──────────────────────────────────────────────────────────┐
│  投资部 / Shop                          [刷新 💰5]        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐        │
│  │ epic │  │传奇  │  │卡包 1│  │卡包 2│  │卡包 3│        │
│  │ 单卡 │  │ 单卡 │  │      │  │      │  │      │        │
│  │ 💰12 │  │ 💰27 │  │ 💰 7 │  │ 💰 5 │  │      │        │
│  │      │  │ ✨   │  │      │  │      │  │ ── │        │
│  └──────┘  └──────┘  └──────┘  └──────┘  └──────┘        │
│   Slot A   Slot B    Slot C    Slot D    Slot E          │
└──────────────────────────────────────────────────────────┘
```

| 槽 | 内容 | 出现概率 | 说明 |
|---|---|---|---|
| **A** | 1 张 epic 单卡 | 100% | 部门轮换（R/S/O 各关循环），玩家可见全部数值 |
| **B** | 1 张传奇卡 | **40%** | 未刷出留空显示「暂无传奇卡可挖角」 |
| **C** | 卡包 | 80% | 6 类卡包之一随机 |
| **D** | 卡包 | 60% | 与 C 槽不重复类型 |
| **E** | 卡包 | 40% | 与 C/D 槽不重复类型 |

> **理论期望槽数**：1 + 0.4 + 0.8 + 0.6 + 0.4 = **3.2 槽**实有商品（含 epic + 卡包 1-3 个）。

### 3.2 卡包定义

6 类卡包：

| 卡包 | 池过滤 | 抽取规则 | 💰 |
|---|---|---|---|
| **猎头礼包** | 全部员工卡 · common | 3 张选 1 | 4 |
| **精英礼包** | 全部员工卡 · elite | 3 张选 1 | 8 |
| **服务礼包** | 全部服务卡 | 3 张选 1 | 5 |
| **功能礼包** | 全部功能卡 | 3 张选 1 | 5 |
| **商业洞察** | 全部商业模式（按解锁关卡过滤） | 3 张选 1 | 7 |
| **神秘包** | 跨稀有度，10% 出传奇 | 3 张选 1 | 10 |

> **神秘包传奇保底**：详见 §7.3。

### 3.3 刷新机制

- **成本**：💰 5（不封顶次数）
- **行为**：清空所有未购买商品 + 重新 roll 5 槽
- **不保留**：单个槽位不支持单独锁定（v1.0 不实现 Balatro 风格 lock，留作 v2 扩展）

### 3.4 购买流程

1. 玩家点击商品 → 弹出商品详情 modal（卡牌大图 / 价格 / 效果）
2. 玩家选「买入」→ 💰 扣除 → 商品加入下关初始牌堆 / 商业模式槽位
3. 卡包：买入后立即弹出抽取 modal（3 选 1），选定后卡牌入下关牌堆
4. **不可悔棋**：购买后无法退款；刷新后未买商品永久消失

### 3.5 商品流向

| 来源 | 流向 |
|---|---|
| epic / 传奇单卡 | 下关初始牌堆 |
| 卡包内员工卡 | 下关初始牌堆 |
| 卡包内服务卡 / 功能卡 | 下关初始牌堆 |
| 卡包内商业模式 | 商学院槽位（与商学院槽位上限共享） |

## §4 模块 2：人事部（牌库整理）

### 4.1 功能：升职

选择牌库中 1 张员工卡（含本关已积累的、即将进入下关的全部员工卡），二选一升级：

#### ① 稀有度升级

| 当前稀有度 | 目标稀有度 | 💰 |
|---|---|---|
| common | rare | 10 |
| rare | elite | 15 |
| elite | epic | 20 |
| epic | legendary | **不可升**（传奇仅商店购买）|

升级效果：

- **AP 不变**
- **基础产出 +25%**
- **稀有度边框 / 卡面光效随之升级**
- **职级保持不变**（升职 ≠ 升级，是横向的稀有度跃迁，叙事上是"人才认证"）

#### ② 词缀附加

为该卡 +1 词缀，类型在以下池子随机 3 选 1：

- `→ 右邻 +10%`
- `← 左邻 +10%`
- `↔ 双向 +5%`
- `部门标识 +1`（用于触发同部门 buff）
- `维持费 -1`（仅 elite+ 可选）
- `P5 收割位倾向 +20%`（在 P5 时额外加成）

**成本**：💰 8（固定，不随稀有度变化）

### 4.2 功能：解雇

永久移除 1 张牌库中的卡（含手牌 / 牌堆 / 冷却池中的任一张）。

| 关卡 | 解雇成本 |
|---|---|
| 关 1-3 | 💰 3 |
| 关 4-6 | 💰 5 |
| 关 7+ | 💰 8 |

> **价格随关递增的设计意图**：抑制后期把牌库精简到极致（单一流派），保持牌库的"杂质"以维持决策深度。

### 4.3 限制规则

- **起始 7 张固定起始手牌**：可解雇、可升职
- **每关每张卡只能被操作 1 次**（升职 或 解雇，不能升职后再次升职）
- **传奇卡可解雇不可升职**
- **同关解雇总数上限**：5 张（防过度精简）

## §5 模块 3：商学院（商业模式 / Balatro Joker 类）

### 5.1 定义

**商业模式（Business Model）** = 全局规则修改器，类比 Balatro 的 Joker 牌。买入后**永久占据一个槽位**直到被替换，每月 / 每次结算 / 每关自动触发对应钩子。

### 5.2 槽位上限

| 关卡 | 槽位上限 |
|---|---|
| 关 1-3 | 3 槽 |
| 关 4-6 | 4 槽 |
| 关 7-10 | 5 槽 |

> 满槽时购买需替换；被替换的商业模式**永久弃置**（不进任何回收池）。

### 5.3 商业模式按 hook 分 3 类

#### 5.3.1 `onMonthStart` 月初钩子

| ID | 名称 | 效果 | 稀有度 | 解锁关卡 | 💰 |
|---|---|---|---|---|---|
| BM_01 | 敏捷开发 | 月初抽牌 +1 | 普通 | 1 | 7 |
| BM_02 | OKR 文化 | 月初手牌 ≥6 张时 AP +1 | 普通 | 1 | 7 |
| BM_03 | 降本增效 | 维持费 -20%（向下取整） | 稀有 | 4 | 13 |
| BM_04 | 精益创业 | 每条产线 AP 消耗 -1（最低 1） | 稀有 | 4 | 13 |
| BM_05 | 远程办公 | 月初手牌上限 +2（仅本月）| 普通 | 1 | 7 |

#### 5.3.2 `onSettle` 结算乘区

| ID | 名称 | 效果 | 稀有度 | 解锁关卡 | 💰 |
|---|---|---|---|---|---|
| BM_06 | 平台战略 | 同部门相邻协同基础 +10%（默认 20% → 30%）| 普通 | 1 | 7 |
| BM_07 | 中台架构 | 服务卡 AP -1 | 稀有 | 4 | 13 |
| BM_08 | R 部门倾斜 | R 部门卡产出 +15% | 普通 | 1 | 7 |
| BM_09 | 收割位增益 | P5 收割位倍率 ×1.5 → ×1.7 | 稀有 | 4 | 13 |
| BM_10 | 启动位增益 | P1 启动位 +20% → +30% | 普通 | 1 | 7 |

#### 5.3.3 `onCharge` 一次性 charge

| ID | 名称 | 效果 | 稀有度 | 解锁关卡 | 💰 |
|---|---|---|---|---|---|
| BM_11 | 破坏式创新 | 每关 1 次免冷却复用产线（按钮触发） | 传奇 | 7 | 27 |
| BM_12 | ESG 报告 | 关末 💰 +20%（评级奖励基础上） | 稀有 | 4 | 13 |
| BM_13 | 黑天鹅对冲 | 每关 1 次免维持费（按钮触发） | 稀有 | 4 | 13 |

### 5.4 初始池规模

v1.0 首版落 **13 张**（如上表），后续 milestone 扩展到 20+。

### 5.5 触发顺序与叠加规则

钩子触发顺序（同钩子内按购买顺序）：

```
月初阶段 (Phase 1-4):
  ① 月报 → ② 事件 → ③ 招聘市场 → ④ 抽牌
                                    ↑
                       【onMonthStart 钩子全部触发】

结算阶段 (Phase 6):
  ① 静态 buff 计算（含卡词缀 + 位置加成 + onSettle 商业模式）
  ② 链式触发 P1 → P5
                       【onSettle 商业模式作为乘数同步参与计算】

charge 按钮（玩家手动触发）：
  随时可用，每关充能 1 次，[局结算前] 决策

关末阶段 (Phase 7):
  ① 评级判定
  ② 💰 发放（含 ESG 报告等 onCharge 关末 buff）
  ③ 充能回满（onCharge 类）
```

**叠加规则**：

- 同钩子多张商业模式 → 按购买顺序依次应用
- 乘数效果 → 同区相乘（如 P5 ×1.5 + 收割位增益 ×1.7 → 卡贡献 ≥60% 时直接生效 ×1.7）
- 加法效果 → 同区相加（如 R 倾斜 +15% + 平台战略 +10% → 同部门 R 邻接卡总 +25%）
- 加法与乘法 → 先加后乘

### 5.6 刷新

- **成本**：💰 4（不封顶次数）
- **每次刷出**：3 张商业模式（按当前关卡解锁池过滤）
- **不保留**：单张不支持锁定

## §6 模块 4：董事访谈（随机事件）

### 6.1 触发与流程

- **进入 Hub 时强制弹出 1 个事件 modal**
- 玩家**必须选择 1 项**才能关闭，事件结果以 toast 写到顶栏
- 关闭后正常进入 Hub
- 每场会议 1 个事件（不可跳过、不可刷新）

### 6.2 数据结构

```yaml
event:
  id: EV_BM_01
  title: 核心员工被挖
  trigger: intermission   # 区别于 §B.3 月度事件
  flavor: |
    某竞品猎头以高出 50% 的薪资来挖你最强的研发主管。
    HR 主管把球抛给了董事会：
  options:
    - label: 留人（- 💰）
      cost: { budget: 8 }
      effect: { type: keep, target: highestAP_R }
    - label: 放走
      effect: { type: remove, target: highestAP_R }
    - label: 反挖对方
      cost: { budget: 12 }
      effect: { type: add, pool: rare_R, count: 1 }
```

### 6.3 首版 5 个事件原型

| ID | 名称 | 选项树 |
|---|---|---|
| EV_BM_01 | **核心员工被挖** | ① 留人 -💰8 ② 放走 -1 张最高 AP R 卡 ③ 反挖对方 -💰12 +1 张随机稀有 R 卡 |
| EV_BM_02 | **投资人施压** | ① 妥协（下关 ¥ 目标 +10%）② 顶住 -💰5 ③ 反手画饼 +💰3 但下关首月手牌 -1 |
| EV_BM_03 | **行业大会** | ① 派 R 团队（解锁 1 张随机 R epic 到下关招聘池）② 派 S 团队（同 S）③ 派 O 团队（同 O） |
| EV_BM_04 | **黑天鹅** | ① 顺势抄底 50%：+💰10 / 50%：-💰5 ② 谨慎观望（无变化） |
| EV_BM_05 | **行业并购** | ① 吞并（永久 +1 槽商业模式上限）-💰15 ② 拒绝 +💰3 ③ 观望（无变化） |

### 6.4 触发权重

首版 5 个事件等权随机（20% 各）。后续 milestone 可扩展：
- 加入条件触发（如关卡数 / 当前牌库特征 / 上关评级）
- 加入剧情链（连续 3 关投资人施压 → 触发特殊董事会换届事件）

## §7 平衡设计

### 7.1 商品价格曲线（汇总）

```
关 N 时：
  epic 单卡: 💰 = N + 8        // 关 1: 9; 关 5: 13; 关 9: 17
  传奇单卡: 💰 = N × 4 + 15    // 关 1: 19; 关 5: 35; 关 9: 51
  普通商业模式: 💰 = N + 6     // 关 1: 7; 关 9: 15
  稀有商业模式: 💰 = N + 12    // 关 4: 16; 关 9: 21
  传奇商业模式: 💰 = N + 18    // 关 7: 25; 关 9: 27
```

**预算压力测试**：

| 关 N | 累计 💰 期望 | 顶级商品总价示例 | 余量 |
|---|---|---|---|
| 关 3 末 | 60-72 | epic 11 + 商业模式 9 + 卡包 7 = 27 | 充裕 |
| 关 6 末 | 120-145 | 传奇 39 + 稀有商业模式 18 + 升职 epic 20 = 77 | 紧张 |
| 关 9 末 | 200-235 | 传奇 51 + 传奇商业模式 27 + 升职 20 + 解雇 8 = 106 | 必须刷 |

> 设计推论：**关 9 玩家典型预算 < 关 9 顶级商品总价的 2.5 倍**，保证决策张力。

### 7.2 内容解锁曲线

| 内容 | 解锁关卡 |
|---|---|
| 普通商业模式 | 关 1 |
| 卡包 - 猎头 / 服务 / 功能 | 关 1 |
| epic 单卡 | 关 1 |
| 商业模式槽位 4 | 关 4 |
| 稀有商业模式 | 关 4 |
| 神秘包（10% 传奇） | 关 4 |
| 传奇单卡 40% 概率 | 关 1 |
| 商业模式槽位 5 | 关 7 |
| 传奇商业模式 | 关 7 |

### 7.3 RNG 保底机制

| 情境 | 保底 |
|---|---|
| 传奇卡连续 3 关未刷出 | 第 4 关刷出概率提升至 60%；触发后重置 |
| 神秘包内传奇概率 | 10%（独立 roll，不受卡池保底影响） |
| 商业模式槽位刷重复 | 同槽内 3 张商业模式不重复；与已购买持有的不重复 |
| 卡包内卡牌 | 池内 3 张不重复 |

### 7.4 反滚雪球设计

| 机制 | 作用 |
|---|---|
| 解雇成本随关递增（3→5→8） | 抑制后期单流派精简 |
| 商业模式被替换永久弃置 | 满槽决策需谨慎，避免无脑换 |
| 关末 💰 发放线性（不指数） | 后期通货膨胀有限 |
| 高级商业模式后期才解锁 | 前期攒钱无处用最强档商品 |
| 每张卡每关只能操作 1 次 | 防止人事部连续升职单卡 |

## §11 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 商业模式触发顺序歧义 | 数值不可预测 | §5.5 明确钩子优先级 + 加减乘叠加规则 |
| 💰 全保留导致前期空过 | 节奏崩 | §7.1 价格曲线 + §7.2 内容解锁后期化 |
| 4 模块决策疲劳 | 流程冗长 | 进入下关按钮始终可点 + 事件 modal 自带终止 |
| 传奇卡 40% 极端 RNG | 整局没刷到 | §7.3 连续 3 关未刷出 → 第 4 关 60% 保底 |
| 3 卡包槽位泛滥 | 商店过满 | §3.1 阶梯概率 80% / 60% / 40%，期望 1-2 槽 |
| 升职 + 商业模式叠乘失控 | 后期碾压 | §7.4 升职每关每卡 1 次 / 解雇成本递增 / 商业模式槽位渐进 |
| 事件 modal 强制感过强 | 玩家抵触 | 事件设计为"剧情爽点"非"惩罚"，多数选项有正反馈 |
| 商业模式池过小重复刷 | 决策枯燥 | §5.4 v1.0 13 张已足够 / v1.1 扩至 20+ |
