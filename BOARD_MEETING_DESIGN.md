## §1 概览

### 1.1 定位与设计目标

「董事会会议」是融资阶段晋升时的跨阶段运营环节，对应商业叙事是「上一阶段融资目标达成，董事会召开战略会议，向公司注资，并由 CEO 调整下一阶段的经营战略与资本结构」。

**设计目标**：

1. **资本分配核心**：引入**财务部**，解耦「现金」与「留存利润」，玩家在此决定利润提取比例，进行资本分配。
2. **状态全程连续**：告别旧版的关卡重置制。跨阶段时，玩家的所有卡牌、产线运行状态、已订阅商业模式、累计月份全部保留，只在跨门槛时注入新一轮的 Entry Grant 资金。
3. **牌库深度调整**：通过在人事部升级/解雇卡牌（向社会输送人才），在商学院订阅/退订商业模式，实现牌库的持续进化。

### 1.2 触发与流程

当月度结算后，估值 $V$ 达到下一阶段门槛，且**连续 2 个月达标**时，在月度结算流程结束时自动触发「董事会会议」（Intermission 状态）。

```
月度结算完成 (Valuation 连续 2 月达标)
        │
        ├─ if Stage == 9 → 触发【全游戏终极胜利结算】（行业第一）
        │
        └─ else → 进入【董事会会议】(Intermission)
                    │
                    ├─ 1. 投资人自动注资 (cash += stage.entryGrant)
                    ├─ 2. 强制弹出【董事访谈】事件，玩家必须做 3 选 1 决策
                    ├─ 3. 进入 Board Meeting Hub 自由操作 (财务部/投资部/人事部/商学院)
                    │
                    └─ 玩家点击「进入下一阶段」 → 返回主战斗界面，开启下一阶段
```

## §2 经济与结算流

### 2.1 现金/留存利润解耦

在月度经营中，产线的净利润（`income - burn`）全部流入**留存利润**（`retainedEarnings`）中，玩家在局内的「现金」（`cash`）保持不变。

玩家只能在跨阶段的**董事会会议**中，通过**财务部**将留存利润提取转化为可花现金。

### 2.2 阶段晋升注资 (Entry Grant)

进入董事会会议时，投资人将根据新阶段注入一笔基础资金，直接增加玩家的 `cash`：

| Stage | 阶段名 | 估值门槛 ($V$) | 阶段注资 (¥ cash) |
|---|---|---|---|
| 1 | 天使轮 | 0 | 起步 ¥30 (开局自动注入) |
| 2 | 种子轮 | 300 | +¥25 |
| 3 | A 轮 | 1000 | +¥50 |
| 4 | B 轮 | 2500 | +¥100 |
| 5 | C 轮 | 5000 | +¥200 |
| 6 | D 轮 | 10000 | +¥400 |
| 7 | IPO | 20000 | +¥700 |
| 8 | 千亿 | 40000 | +¥1200 |
| 9 | 行业第一 | 80000 | +¥2000 (直接达成终极胜利) |

## §3 财务部 (Finance Department)

财务部是 v3.2 重构的核心模块，位于董事会主界面的最顶部。

### 3.1 留存利润提取机制

- 进入董事会会议时，财务部显示当前公司的「留存利润余额」以及提取滑块。
- 玩家可选择提取比例：**0% / 30% / 60% / 100%**。
- 确认提取后：
  - `cash += Math.floor(retainedEarnings * ratio)`
  - `retainedEarnings -= Math.floor(retainedEarnings * ratio)`
- **注意**：提取操作在每场会议中**仅限一次**。一旦点击“确认提取”滑块即锁定，防止多次提取或微调套利。

## §4 各运营部门

### 4.1 投资部 (Investment Shop)

投资部用于招募高端卡牌和开辟人才礼包，共 5 个商品槽位，使用 **¥ 现金** 结算：

```
┌──────────────────────────────────────────────────────────┐
│  投资部 / Shop                          [刷新 ¥5]         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐        │
│  │ epic │  │传奇  │  │卡包 1│  │卡包 2│  │卡包 3│        │
│  │ 单卡 │  │ 单卡 │  │      │  │      │  │      │        │
│  │  ¥12  │  │  ¥27  │  │  ¥7  │  │  ¥5  │  │      │        │
│  │      │  │ ✨   │  │      │  │      │  │ ──   │        │
│  └──────┘  └──────┘  └──────┘  └──────┘  └──────┘        │
│   Slot A   Slot B    Slot C    Slot D    Slot E          │
└──────────────────────────────────────────────────────────┘
```

- **Slot A**：1 张史诗 (epic) 单卡，100% 出现，价格约为 `¥(Stage + 8)`。
- **Slot B**：1 张传奇 (legendary) 单卡，40% 概率出现。价格约为 `¥(Stage * 4 + 15)`。
- **Slot C-E**：各类卡包（3选1），价格在 `¥4` - `¥10` 之间。
- **刷新 (Roll)**：每次消耗 `¥5`。

### 4.2 人事部 (HR Office)

人事部提供对牌库中现有卡牌的管理，主要包括**升职**和**解雇**（向社会输送人才）。

#### ① 升职 (Card Upgrade)
升级卡牌的稀有度，提升基础产出（AP 消耗保持不变）：
- **普通 (common) → 稀有 (rare)**：消耗 `¥10`
- **稀有 (rare) → 精英 (elite)**：消耗 `¥15`
- **精英 (elite) → 史诗 (epic)**：消耗 `¥20`
- **史诗 (epic) → 传奇 (legendary)**：不可直接升级（传奇仅商店产出）

#### ② 解雇 (Card Dismiss)
永久从公司牌库中移除某张卡牌（包括手牌、牌堆或冷却池）。
解雇成本随着公司经营阶段的上升而提高，防止后期无脑精简牌库：
- **阶段 1-3**：每次解雇消耗 `¥3`
- **阶段 4-6**：每次解雇消耗 `¥5`
- **阶段 7+**：每次解雇消耗 `¥8`
- **限制**：每场会议「向社会输送人才」上限为 5 张，且已被升职的卡在当场会议不可再被解雇。

### 4.3 商学院 (Business School)

商学院用于购买和配置**商业模式 (Business Model, BM)**（全局乘区与规则修改器）。

- **订阅 (Subscribe)**：购买时需支付一次性签约费（¥），订阅后该 BM 占用一个槽位，且每月结算时自动扣除其对应的 **订阅月费 (bmMonthlyCost)**。
- **槽位上限**：随阶段推进解锁。Stage 1-3 为 3 槽；Stage 4-6 为 4 槽；Stage 7+ 为 5 槽。
- **退订 (Unsubscribe)**：玩家可在董事会中**免费退订**任何已订阅的商业模式，腾出槽位。退订后该模式被移除，下月起停止扣除月费。
- **刷新 (Roll)**：每次消耗 `¥4` 重新刷出 3 张候选 BM。

### 4.4 董事访谈 (Boardroom Talks)

入场强制弹出事件。玩家通过 3 选 1 抉择确定下一阶段的战略调整：
- 选项包含：支付现金留人、放走卡牌获得少量补偿、增加下一阶段的挑战门槛但换取现金等。
- 事件产生的消耗扣除自 `cash` 余额。

## §5 数值与平衡设计

### 5.1 RARITY_TABLE 数值集中参数

卡牌与商业模式的 Burn 和资产价值在 `cards.js` 的 `RARITY_TABLE` 中统一管理：

| 稀有度 (Rarity) | 卡牌月消耗 (baseBurn) | 上线当月加扣 (extraBurn) | 卡牌估值资产 (assetValue) | BM月度订阅费 (bmMonthlyCost) | BM估值资产 (bmAssetValue) |
|---|---|---|---|---|---|
| **common** | 1 | 0 | 5 | 2 | 8 |
| **rare** | 2 | 1 | 15 | 4 | 25 |
| **elite** | 3 | 1 | 30 | 6 | 50 |
| **epic** | 4 | 2 | 50 | 8 | 80 |
| **legendary** | 7 | 4 | 150 | 14 | 240 |

### 5.2 估值 $V$ 公式

董事会中的各项操作（如买卡、升职、退订等）会即时影响估值 $V$ 的计算：
$$V = \text{PE 估值} + \text{资产估值} + \text{现金溢价}$$
- $\text{PE 估值} = \max(0, \text{最近 3 个月平均净利润} \times 20)$
- $\text{资产估值} = (\sum \text{卡牌资产价值} + \sum \text{商业模式资产价值}) \times 0.5$
- $\text{现金溢价} = \max(0, \text{现金 cash}) \times 0.3$

- **财务部滑块**：以显眼横条形式位于顶部，在玩家未进行提取前呈高亮激活状态，提取后变为灰色锁定状态，展示提取的 ¥ 现金与剩余的留存利润。
- **商学院卡片退订**：已订阅的商业模式卡片下方展示有 `退订` 按钮，点击后立即无代价腾出槽位并免除未来的订阅费。
- **人事部操作确认**：升职与解雇提供二次弹窗确认，人事部列表支持搜索或分类筛选卡牌。
