## §2 技术栈与渲染管线

### 2.1 引擎选型

| 选型 | 决策 | 理由 |
|---|---|---|
| **游戏引擎** | **Phaser 3.x**（最新版） | 2D/2.5D 卡牌游戏甜蜜点；社区成熟；适配 Web/移动端 |
| **渲染模式** | **WebGL**（非 Canvas） | 粒子/着色器/混合模式全套支持 |
| **物理引擎** | Arcade Physics（轻量） | 卡牌掉落/弹跳/碰撞够用，无需 Matter.js 重量级 |
| **粒子系统** | Phaser ParticleEmitter | 链式爆分、星光、烟花 |
| **音频** | Phaser Sound + Tone.js（备选） | 链式音调升 Key 需要 Tone.js 程序化合成 |
| **状态管理** | 自研 EventBus + StateMachine | 战斗有明确 Phase 状态机，不需要 Redux |
| **UI 框架** | Phaser 原生 + 少量 DOM 叠加 | Tooltip、详情面板等用 DOM 更灵活 |

### 2.2 2.5D 实现方案

**核心技术：伪 3D 通过透视投影 + Z 轴叠层 + 投影阴影实现**

```
视觉层级（从远到近，Z 轴递增）：
┌────────────────────────────────────┐
│ Layer 0  背景天际线（远景渐变）       │
│ Layer 1  办公楼远景（视差滚动）       │
│ Layer 2  办公室地板（斜视角网格）     │
│ Layer 3  产线槽位（高出地板，有厚度） │
│ Layer 4  卡牌阴影（落在产线上）       │
│ Layer 5  卡牌本体（可悬停浮起）       │
│ Layer 6  特效层（粒子、光线、连线）   │
│ Layer 7  UI 层（资源条、按钮）        │
│ Layer 8  弹字层（数字爆出）           │
│ Layer 9  全屏特效（震屏、闪光、暗角） │
└────────────────────────────────────┘
```

产线"槽位"用**等距投影（isometric）**绘制，给玩家"工位/办公桌"的真实感。

### 2.3 性能指标

| 指标 | 目标 |
|---|---|
| 目标帧率 | 60 FPS 稳定 |
| 链式爆分时屏幕粒子数 | ≤ 500 |
| 单次结算总时长 | 3-5 秒 |
| 首屏加载 | ≤ 3 秒（含核心资源）|
| 移动端最低适配 | iPhone XR / 中端安卓 |

### 3.3 关键区域规格

#### 产线槽位

- 每个槽位尺寸：240 × 320 px（卡牌标准尺寸）
- 槽位间距：20 px
- 产线总宽：240×5 + 20×4 = 1280 px
- 槽位有"工位桌面"的等距视觉，**带 4-8 px 阴影深度**

槽位状态视觉：

| 状态 | 视觉 |
|---|---|
| 空槽（可布置）| 半透明虚线边框，hover 时高亮 |
| 已放置（未锁定）| 卡牌实体，可拖出 |
| 已锁定（生产中）| 卡牌发淡光，无法移动，显示"工作中"角标 |
| P1 启动位 | 槽位地面有橙色光圈 |
| P3 中枢位 | 槽位有蓝色脉冲光晕 |
| P5 收割位 | 槽位有金色边框 |

#### 手牌区

- 卡牌默认尺寸：160 × 224 px（小于产线槽位中的卡，节省空间）
- 手牌 10 张时自动收缩，**类似炉石**的扇形展开
- hover 单张卡 → 放大到 240×320 + 上浮 60px + 显示完整数值/词条

#### 牌堆和冷却池

- 牌堆图标：卡牌背面 + 角标数字
- 冷却池图标：休息小屋图标 + 当前冷却中卡数
- 点击展开：弹出 DOM 浮窗，列出所有卡 + 各自状态
- 牌堆顺序不可见（保留 roguelike 抽牌惊喜）
- 冷却池可见（玩家需要规划"谁什么时候回来"）

## §4 美术风格

### 4.1 整体风格定位

> **复古像素风 × 现代特效 × 商务办公主题**

参考游戏：
- 《Stardew Valley》——像素风的角色和环境
- 《Balatro》——卡牌质感 + 高对比度爆分演出
- 《Dave the Diver》——2.5D 像素 + 流畅过场
- 《Hades》——动作流畅 + 强烈视觉反馈

### 4.3 像素分辨率

- **卡牌图标**：64×64 px 像素艺术
- **角色立绘**（员工 / 高管）：96×128 px
- **办公室场景元素**（桌椅、电脑、绿植）：32×32 ~ 128×128 px tile
- **UI 图标**（资源、按钮）：32×32 px

像素绘制时**严格对齐网格**，但**动画运动**允许亚像素插值（保证流畅）。

### 4.4 字体规范

| 用途 | 字体 | 大小 |
|---|---|---|
| 数字（资源、产出）| **Press Start 2P** 或自制像素数字 | 16-48 px |
| 卡牌标题 | **Pixel Operator Bold** | 14 px |
| 卡牌描述 | **思源黑体 Medium** | 11 px |
| 弹字（爆分）| **Bungee** 或加粗描边像素字 | 60-120 px |
| 系统 UI | **思源黑体** | 12-16 px |

中文部分用思源黑体（清晰），数字和英文用像素字（风味）。

#### 4.5.2 六个分区的详细规范

**A. 顶部信息条（三栏并列）**

| 栏 | 内容 | 视觉 |
|---|---|---|
| 左 | **部门 / 牌类别**徽章 | 部门色背景 + 部门图标（如 "研发 R"、"销售 S"、"运营 O"、"功能"、"服务"、"传奇"）|
| 中 | **稀有度**标签 | 稀有度色 + 中文（"普通"/"稀有"/"精英"/"史诗"/"传奇"）|
| 右 | **AP 消耗** | ⚡ 图标 + 大字数字（玩家最常看的数据，要醒目）|

**注**：卡牌本身不显示价格，购买和招聘价格在董事会会议的各个部门中展示。

**B. 角色立绘区（主视觉）**

- 占卡面最大区域（约 38% 卡高），让玩家**一眼记住"这是谁"**
- 像素艺术风格，96×128 ~ 128×160 px
- 立绘背景按稀有度差异化（见 §4.5.3）
- 立绘需有**人物特征**——程序员戴眼镜、销售西装领带、运营 Excel 表格等

**C. 卡名小标题**

- 单行居中显示卡名（如"研发专员"、"销售总监"、"市场调研"）
- 字号约 14px，**比顶部信息条略大，比数值区小**
- 字体粗细：中粗（让玩家秒认）
- 颜色按稀有度调整：
  - common 白色
  - rare 浅蓝
  - elite 紫色
  - epic 橙色 + 微光
  - legendary 金色 + 闪光

**D. 产出数值 / 功能区（核心数据）**

按卡类型展示不同内容（**大字号 + 高对比度**）：

| 卡类型 | 展示内容 | 示例 |
|---|---|---|
| 员工卡 | 💵 基础产出 ¥+ 波动标记 | `💵 22 ↑` |
| 功能卡 | 主要 buff 效果 | `↔ +25%` 或 `相邻 R x2` |
| 服务卡 | 主要全局效果 | `本月免维持费` |
| 传奇卡 | 标志性效果摘要 | `全线 xMult x1.5` |

数值波动颜色标记详见 §4.6。

**E. 缩略附加效果区（hover 展开）**

默认状态：
- 显示效果**首行简略文本**，超出截断为 `…`
- 多条效果用"·"分隔
- 字号比 D 区小（约 11px）

示例：
```
默认显示：  → 右邻 +12% · P3 时 +25% …
hover 展开：→ 右邻 +12%
            P3 时自身 +25%
            同部门相邻 +20%
            【词条·完美主义】
            【词条·收割者】
```

hover 触发动画：卡面上浮 60px + 放大到 240×320 + 平滑展开效果区（0.2s）。

**F. Flavor 文本区**

- 斜体灰色字
- 字号 11px
- 单行（超长截断 `…`），hover 后完整显示
- 内容是吐槽/俏皮话，增加角色个性

#### 4.5.3 稀有度的视觉差异化（装备化系统的关键）

| 稀有度 | 卡边框 | 顶部条背景 | 立绘背景 | 整体特效 |
|---|---|---|---|---|
| **common 普通** | 灰色单线 | 灰底 | 纯色背景 | 无 |
| **rare 稀有** | 蓝色双线 | 蓝渐变 | 微光底 | 无 |
| **elite 精英** | 紫色 + 边角闪光 | 紫渐变 | 紫色渐变 + 粒子 | 立绘背后慢速粒子 |
| **epic 史诗** | 橙色 + 持续脉冲 | 橙渐变 + 火焰描边 | 火焰般背景 | 立绘有金色光圈 + 脉冲 |
| **legendary 传奇** | 金色 + 旋转光带 | 金渐变 + 太阳光放射 | 太阳光放射 | 立绘有完整循环动画 + 持续闪光 |

玩家**一眼就能看出**这张卡的稀有度。

### 4.6 数值波动的视觉表达

玩家开出一张卡时，要让他看到**数值的"装备化"感**：

| 偏离程度 | 颜色 | 标记 | 示例 |
|---|---|---|---|
| ≥+15% 高于中位数 | **亮绿** | ↑↑ | `💵 25 ↑↑` |
| +5% ~ +15% | **绿色** | ↑ | `💵 22 ↑` |
| ±5% 中位数附近 | **白色** | 无 | `💵 20` |
| -5% ~ -15% | **橙色** | ↓ | `💵 17 ↓` |
| ≤-15% 低于中位数 | **红色** | ↓↓ | `💵 14 ↓↓` |

**示例对比**：
- 研发专员基础产出中位数 20
- 这次开出 25 → 显示 `💵 25 ↑↑`（亮绿，玩家心中暗喜）
- 那次开出 14 → 显示 `💵 14 ↓↓`（红色，玩家想换一张）

**这就是"刷装备"的兴奋感来源**。

│      ╚═══════════════╝              │

词条规则：
- **大词条**：紫色实线框，左上角带"★"标记
- **小词条**：紫色虚线框，无标记
- 词条名后**hover 显示具体效果文本**（避免卡面太挤）
- 词条用**特殊字体或加粗**，与普通效果文本区分

> **注**：词条区独立于 E 缩略附加效果区，位于其下方，仅 elite/epic/legendary 卡显示。普通/稀有卡无此区域，flavor 文本直接接在 E 区之后。

### 4.8 招聘与升级价格显示

本游戏移除了战斗界面的月度招聘市场。所有的招聘和升级均在跨阶段的**董事会会议**中进行。

- 招聘价格在董事会投资部（Shop）展示，由提取的 ¥ 现金支付。
- 升职和解雇价格在董事会人事部（HR Office）展示。
- 订阅和退订价格在董事会商学院（Business School）展示。

## §5 交互流程：从月初到月末

> 完整每月流程见 GAME_DESIGN_FOUNDATION §A.5。本节描述**每一步玩家看到什么、动画如何呈现**。

### 5.1 Phase 1 - 月初报到（自动，2-3 秒）

**画面**：场景从黑屏淡入到办公室。日历翻页（左上角"第 3 月"动画），月份数字弹跳。

**演出顺序**：

1. **冷却员工归来动画**（如有）
   - 冷却池图标"弹"一下 + 抖动
   - 卡片从冷却池飞出，画弧线飞向牌堆图标
   - 牌堆数字 +N，闪一下
   - 同时音效：清脆"咔哒"

2. **维持费扣除**（如有）
   - 屏幕中央飘起红色"-¥1 维持费"，向 ¥ 资源条飞去
   - ¥ 资源条数字滚动减少
   - 音效：低沉"咚"

3. **产线工作进度推进**
   - 在职员工卡牌左上角的"剩余 N 月"数字 -1
   - 数字跳动 + 微震

**总耗时**：1.5 - 3 秒（取决于回归员工数量）

### 5.2 Phase 2 - 事件揭示（中断式，4-6 秒）

**画面**：所有 UI 暗化（50% 黑色遮罩），中央弹出事件卡。

**事件卡视觉**：

```
┌──────────────────────────┐
│                          │
│   ★ 本月事件 ★          │
│                          │
│   【天使融资到账】        │
│                          │
│   [事件插画 - 像素风]     │
│                          │
│   你的天使投资人对项目    │
│   非常满意，决定追投。    │
│                          │
│   📈 ¥ +10               │
│   📊 关卡末累计 ¥ 目标 +50│
│                          │
│      [接受 → ]           │
│                          │
└──────────────────────────┘
```

**演出**：

1. 屏幕暗化 + 卡牌从下飞上 0.4s（弹性 easeOutBack）
2. 标题打字机效果 0.5s
3. 插画淡入 0.3s
4. 描述文字打字机 1-2s
5. 效果数字逐个弹出（每个 0.2s）
6. 按钮 hover 闪光
7. 玩家点击 → 卡牌爆碎 + 效果飞向对应资源条 0.5s

**总耗时**：约 4-6 秒（玩家阅读时间不限）

### 5.3 Phase 3 - 抽牌（自动，2-3 秒）

**画面**：从牌堆飞出 3 张卡到手牌区。

**演出**：

1. 牌堆图标弹跳 + 闪光
2. 3 张卡背朝上从牌堆飞出，**呈扇形依次飞到手牌位置**（每张延迟 0.15s）
3. 每张卡飞到位后翻面（0.3s 翻转动画），显示正面
4. 翻面后**自动 roll 数值的提示**——数字从 0 滚动到实际值
5. 音效：每张抽出时"sw" + 翻面时"啪"

**手牌超 10 张处理**：

1. 弹出"手牌超出上限"提示
2. 进入**强行弃牌模式**：玩家点击要弃的卡，被点击的卡飞向弃牌堆（消失）
3. 弃够数量后自动继续

### 5.4 Phase 4 - 产线布置（核心决策阶段）

> **这是玩家投入时间最长的环节**，必须做到顺手 + 实时反馈。

#### 5.4.1 拖拽机制

- **抓取**：长按或鼠标按下 0.1s
- **拖动**：卡牌跟随光标，**有 60% 透明残影留在原位**
- **吸附**：拖到产线槽位附近时，槽位高亮闪光，卡片吸附进入
- **放下**：松开光标，卡片落入槽位（弹性 easeOutBack 0.3s）
- **AP 检查**：拖到槽位时实时显示"如果放置 AP 将剩余 X"，AP 不足时槽位**变红 + 摇晃**

#### 5.4.2 实时预览

**关键**：每次卡牌进入/离开产线，**右侧栏实时计算并显示**本月预估产出：

```
┌──────本月预估──────┐
│                    │
│  产线 A：          │
│   预估 ¥ 收入: 220 │
│                    │
│  产线 B：          │
│   预估 ¥ 收入: 0   │
│   （未布置）        │
│                    │
│  本月总计:         │
│   ¥ +220           │
│                    │
└────────────────────┘
```

数字**滚动更新**（200ms），不是瞬间跳变。

#### 5.4.3 邻接协同高亮

- 鼠标 hover 一张已放入产线的卡 → 周围与之协同的卡之间**画发光连线**
- 连线颜色按效果类型：
  - 同部门相邻：金色虚线
  - 方向 buff（→/←）：箭头流动光线
  - 功能卡效果范围：脉冲圆环
- hover 离开 → 连线渐隐 0.3s

#### 5.4.4 撤销

- 卡牌可自由拖出产线、调换顺序
- **结算锁定前**所有操作可逆
- 提供"清空产线"按钮（带二次确认）

#### 5.4.5 确认结算按钮

- 玩家满意后点击"▶ 开始结算"
- **二次确认弹窗**：显示本月预估 + 提示"产线锁定后无法修改"
- 确认后进入 Phase 5

### 5.5 Phase 5 - 激活结算（**核心爽点**）

> 整个游戏的**情绪最高潮**。每一帧都要精心设计。

#### 5.5.1 锁定前奏（1 秒）

1. UI 暗化（30% 黑色遮罩），手牌、按钮淡出
2. 摄像头**缓慢拉近**到产线（zoom 1.0 → 1.15）
3. 产线槽位**逐个亮起**（从 P1 到 P5），每个延迟 0.1s
4. 音效：低沉的蓄力声（"嗡"），音调持续升高

**画面感**：像格斗游戏的"FIGHT" 喊出前的瞬间。

#### 5.5.2 静态 buff 计算可视化（1.5 秒）

**目的**：在激活前让玩家"看到自己设计的协同生效了"。

1. 所有协同关系**画发光连线**（同时浮现）：
   - 同部门相邻 → 金色弧线
   - 方向 buff → 箭头流动
   - 功能卡范围 → 脉冲圆
2. 每条连线尽头出现小标签："+10%" "x2" 等
3. 受影响卡牌的**基础数字闪烁**，最终更新为加成后的值
4. 音效：每条连线触发时"叮" + 连续合奏

**画面感**：像电路图通电的瞬间，节点连接，能量贯穿。

#### 5.5.3 链式激活（**核心动画**，2-3 秒）

**这是整个游戏最重要的演出**。设计配方如下：

**节奏**：P1 → P2 → P3 → P4 → P5 依次激活，**间隔递减**（前慢后快）：

```
P1 激活 → 等 0.35s
P2 激活 → 等 0.30s
P3 激活 → 等 0.25s
P4 激活 → 等 0.20s
P5 激活 → 等 0.10s
最终爆发
```

总耗时约 1.2 秒（含激活动画）。

**单卡激活动画**（每张 0.4s）：

| 时间 | 效果 |
|---|---|
| 0.00s | 卡牌**强烈震动**（位移±4px，5 次往复）|
| 0.05s | 卡牌**爆发光晕**（部门颜色发光圆，扩散 0.3s）|
| 0.10s | 卡牌**弹跳上浮 30px**（弹性曲线 easeOutBack）|
| 0.15s | 产出数字弹出（白色像素字，120% 大小），向上飘 |
| 0.25s | 数字开始**飞向中央汇总点** |
| 0.30s | 飞行轨迹拉出**部门颜色拖尾粒子** |
| 0.40s | 卡牌落回原位，光晕消散 |

**链式传递效果**：

- 当 P1 激活后，**光线沿着产线**向 P2 流动（速度 800 px/s）
- 光线到达 P2 时触发 P2 激活
- 光线颜色：白色 → 渐变到部门色 → 渐变到下一个部门色
- 视觉上：像多米诺骨牌 + 闪电传导

**音效配方**：

- 每张卡激活音调依次升高（C4 → D4 → E4 → G4 → C5）
- 同时音量逐步增加（0.5 → 0.6 → 0.7 → 0.8 → 1.0）
- 背景从无声 → 低沉嗡嗡 → 鼓点加快
- **最后一张激活时**：钢琴猛击 + 玻璃破碎音

#### 5.5.4 数字爆破（**最爆点**，0.8 秒）

P5 激活完成后：

1. **全屏闪白**（0.1s 100% 白色覆盖，快速消退）
2. **屏幕震动**（amplitude 12px，0.3s 衰减）
3. **径向模糊**从中央向外（0.3s）
4. **总产出数字砸下来**：
   - 从屏幕顶部高速坠落
   - 字体超大（120px）
   - 部门混合色描边
   - 落到屏幕中央时**冲击波动画**（圆环扩散）
5. **粒子爆炸**：以数字为中心，喷射 200+ 金色粒子，向四周散开
6. **数字本身有粒子拖尾**

**音效**：

- 闪白瞬间：高频"嘶"
- 数字落下：弦乐 glissando（滑音）
- 落定瞬间：低音爆破鼓（808 kick）
- 后续 0.5s：金币撒落声 + 欢呼声

**画面感**：像格斗游戏的 PERFECT! / Balatro 的爆分瞬间，**最大化情绪释放**。

#### 5.5.5 资源累计（1 秒）

爆破完成后：

1. 结算数字与产线卡牌产出的文字和数值变白变大，没有繁杂的转换，直接飞向 ¥ 资源条，同时在过关结算画面中文字与数值变白变大，去掉“重新开始”按键。
2. ¥ 资源条数字滚动增加（同时累计 ¥ 进度条也增长）
3. 资源条**闪烁绿光**
4. 如果累计 ¥ 跨过关卡目标线 → 关卡目标显示打勾 + 全屏金光闪烁 + 喝彩音效
5. 音效：金币入袋"叮叮叮"

> v0.3 后 ¥ 是唯一的收入资源，不再有 ⭐ 估值分裂转化。

### 5.6 Phase 6 - 月末（自动，1 秒）

- 产线已激活 of 卡牌**下沉**（如果工作完成）→ 飞到冷却池
- 冷却池数字 +N
- 屏幕过渡黑屏 0.3s
- 进入下一月（如果达到过关门槛则触发阶段晋升）

## §6 阶段晋升与董事会会议（跨阶段 Intermission）

### 6.1 阶段达标与晋升触发

在月度结算完成后，如果公司的估值 $V$ 跨过了当前阶段的晋升门槛（例如种子轮的 $V \ge 300$ 门槛），在本轮（月）结束后直接触发阶段晋升：
1. **晋升过关结算画面**：文字与数值变白变大，显示当前估值、累计经营月份。
2. **按钮**：去掉“重新开始”按键，仅保留“进入董事会”按钮。

### 6.2 董事会会议（Board Meeting Intermission）

点击“进入董事会”后，进入半透明白色（50%透明度）遮罩下的董事会会议界面。
董事会会议包含以下部门/区域：
1. **财务部（Finance Department）**：选择留存利润（Retained Earnings）的提取比例 (0% / 30% / 60% / 100%)，将累积的利润转为可支配的 **¥ 现金**。提取操作在每场会议中仅限一次。
2. **投资部（Shop）**：玩家可以使用从“留存利润”中提取出的可支配 **¥ 现金** 购买新卡牌包。
3. **人事部（HR Office）**：进行卡牌升级/升职，或使用免费的解雇逻辑 `dismissCardInBoardMeeting` 移去卡牌（向社会输送人才）。
4. **商学院（Business School）**：已习得的商业模式（BM）卡展示与本期开课的商业模式选择。商学院槽位均为正方形，文字说明显示在正方形槽位下方。
5. **董事访谈（Boardroom Talks）**：入场强制弹出事件，通过 3 选 1 决策进行战略调整。

### 6.3 阶段晋升与终极胜利屏

当玩家达成第9阶段（Stage 9，估值达 80,000+）时，触发终极胜利屏幕，展示公司的最终估值、历经的月数、累积的卡牌资产等，宣告达成行业第一的目标。

**演出**：
- 屏幕色调变冷（去饱和度）
- 关卡背景音乐淡出
- 终极胜利文字打字机效果
- 解锁内容用金色高亮

class CardInstance {

## §12 给实现 AI 的额外提示

> 当 Cursor / Claude / Codex 据此文档实现时：

1. **不要追求一次性写完所有功能**。按 §10 优先级分阶段实现。
2. **链式激活和数字爆破是灵魂**。即使其他都简陋，这两个动画必须做到极致。

5. **音效如果一时找不到合适资源，先用占位声**（Phaser 内置或 freesound.org）。
6. **数值波动 roll 时机**：在 §3.2 已明确，请严格执行。
7. **AP / ¥ (现金与留存利润) 资源的图标和颜色规范**见 §4.2，不要混用。⭐ 估值系统已在 v0.3 删除，💰 战略预算在 v3.2 已被彻底移除。
8. **卡牌布局严格按 §4.5 实现**——六分区结构，去除了卡面价格显示，立绘是主视觉。
