# Boss / 竞争公司系统：市场份额对决

> 配套设计文档：`design.md`（设计语言）、`GAME_DESIGN_FOUNDATION.md`、`BATTLEFIELD_DESIGN.md`、`BOARD_MEETING_DESIGN.md`、`CARDS_DESIGN.md`。
> 本文件定义年度对手（boss）系统的玩法、数值、UI 与实施路径。

---

## 0. 概述

每年一个中等难度**竞争公司**，玩家用月收入抢市场份额、把对手压到 ≤ 20% 即获胜。

**设计原则锚点**：
- **死亡频繁但不廉价**：输了不立即 game over，可透支交「划道费」续命，代价大到下个对手前必须重建经济。
- **随机强但不剥夺决策**：对手 archetype 随机，但 3 月预告期间档案全公开（弱点、月收入预估、特殊技能），玩家有充足时间调卡组。
- **规则短，组合深**：核心只有一条规则——「你的月收入 vs 对手月收入 → 市场份额此消彼长」。组合深度从 archetype × 玩家卡组 × 阶段时机 涌现。
- **6 月软上限**：不设硬时限以保留节奏自由，但 6 月内未胜即取消（无奖励无惩罚），逼玩家要么 all-in 要么放弃。

## 1. 核心机制：市场份额对决

### 1.1 年度节奏

| 月份（年内相对） | 阶段 | 说明 |
|------|------|------|
| 第 9 月 | **预告** | 对手档案公布：archetype、预估月收入、特殊技能。倒计时 3 月 |
| 第 10–12 月 | **备战** | 正常月回合，玩家可调卡组/产线/BM。市场份额未启动 |
| 第 13 月起 | **对决开始** | 市场份额条出现，初始 50% / 50% |
| 第 13–18 月 | **对决进行中** | 每月正常回合 + 份额结算 + 对手 archetype 主动效果 |
| 第 19 月起 | **结算/取消** | 玩家胜 / 输 / 6 月超时取消 |
| 第 19–24 月 | **恢复期** | 下个对手预告月（第 21 月） |

5 个对手覆盖 ~60 月一局：tier 1 → tier 5 难度递增，对应当前 MAJOR_EVENTS 的 tier 系统。

### 1.2 份额结算公式（每月）

```
incomeShareDelta = K × (yourIncome − rivalIncome) / (yourIncome + rivalIncome)
yourShare        += incomeShareDelta
rivalShare       −= incomeShareDelta
```

- **K = 10**（每月最多 ±10 个百分点，需 4–6 月才能拉满胜利线）→ 既不会一月结束、也不会拖到超时
- `rivalIncome` = 基础值（按 stage 缩放） × archetype 修正 × tier 倍数
- 初始 50/50，玩家 ≥ 80% 即胜利

### 1.3 三种结果

| 结果 | 条件 | 后果 |
|------|------|------|
| **胜利** | 玩家份额 ≥ 80%（连续 1 月达成） | 见第 5 节"奖励" |
| **输掉** | 玩家份额 ≤ 0% | 强制透支「划道费」= `2 × 当前阶段阈值` 现金。能付则继续游戏（无奖励 + 下个对手延后 6 月 + 估值 ×0.7），不能付则 game over |
| **超时取消** | 6 月对决期内份额始终在 0–80% 之间 | 无奖励无惩罚，对手"撤离"，下个对手按原节奏来 |

## 2. 对手 Archetype（4 种 + 1 终极）

每年随机抽 1 个，同一局不重复，第 5 个为压轴 boss-rival。

| Archetype | 月度行为 | 弱点（玩家应对方向） |
|------|------|------|
| **价格屠夫** | 自身收入低，但每月让玩家 S 部门产出 −8%（累积可叠 5 次） | 加强 R/O 部门、买入"品牌"BM 抵消 |
| **挖角狂魔** | 对决期每月玩家招聘市场 −1 张可选 + 每月 burn +15%（高薪挽留压力） | 备战期囤够低 burn 卡组、提前买好关键 BM |
| **山寨大王** | 对决期每 2 月让玩家最高产出 BM 效果 −25%（不复制、只削弱） | 在备战期把 BM 切换成低显高隐型 |
| **资本壁垒** | 月收入极高（基础 ×1.4），但无主动技能，纯比拼经济硬度 | 必须靠备战期堆产线满载、AP 全花在 R&S 输出 |
| **🏆 终极对手（Y5）** | 综合体：招聘 −1 + BM −25% + 收入 ×1.2 + 每月 −1 AP | 通关条件——需 4 个前置胜利累积的奖励才稳赢 |

档案在第 9 月公布时**完整可见**，包括预估月收入数字。明牌备战。

## 3. 玩家对决期主动操作（每月新增）

对决期每个回合，玩家除正常部署外，可额外进行 1 次「**竞争行动**」（消耗 cash 或 AP，可跳过）：

| 行动 | 消耗 | 效果 |
|------|------|------|
| **价格战** | −20% 本月利润 | 本月份额结算 K 从 10 → 15（爆发） |
| **品牌投放** | 现金 −300 | 本月对手收入 −20%，仅本月 |
| **挖人** | 2 AP + 现金 −500 | 屏蔽对手 1 个 archetype 技能效果，本月失效 |
| **跳过** | 0 | 本月不主动 |

设计意图：让对决期每月都有**额外的 30 秒决策**——比常规月增加一个非部署的高密度选择，自然落地"30 秒一选择"的最小可行版。

## 4. 对手收入公式（关键平衡点）

对手不真的"模拟一局游戏"，用一个轻量公式：

```
rivalBaseIncome   = stageThreshold[currentStage] × 0.6
rivalTierMult     = 1 + 0.15 × (tier − 1)        // tier 1 = 1.0, tier 5 = 1.6
rivalArchetypeMul = {价格屠夫: 0.85, 挖角: 1.0, 山寨: 1.0, 资本壁垒: 1.4, 终极: 1.2}
rivalIncome       = rivalBaseIncome × rivalTierMult × rivalArchetypeMul × monthDrift(±10%)
```

`monthDrift` 是月间小波动，避免完全静态。

后续可在 `simulation.js` 跑 100 局调 K 和系数让胜率维持在 **50–60%**（中等难度）。

## 5. 奖励设计（驱动下一局）

### 5.1 核心奖励：收购对手团队

**胜利即"收购对手团队"，获得对手卡池中随机 3 张卡**直接进入玩家牌库（跳过冷却，下月起可部署）。每个 archetype 关联一个特色卡池：

| Archetype | 特色卡池（胜利后随机抽 3） |
|------|------|
| 价格屠夫 | 偏低 burn 高产出的 O/S 部门卡 + 1 张"价格战" BM |
| 挖角狂魔 | 高 tier 高 burn 的明星 emp 卡 + 1 张"猎头网络" BM |
| 山寨大王 | rare/elite 级 fun 卡 + 1 张"反向工程" BM |
| 资本壁垒 | 高 baseOutput 稳定型 emp 卡 + 1 张"资本护城河" BM |
| 终极对手 | 必给 1 张 legendary + 2 张随机 elite/epic |

### 5.2 附加双轨奖励

| 击败对手 | 本局即时 buff | 元解锁（持久化存档） |
|------|------|------|
| 1 号 | 永久 +5% 收入 | 新职业起手卡 1 张 |
| 2 号 | 永久 burn −5% | 下局 +500 起始现金 |
| 3 号 | 招聘市场每月 +1 张 | 解锁该 archetype 出现概率（更多重玩多样性） |
| 4 号 | 永久 CCR +5% | 下局可跳过第 1 个对手 |
| 5 号（终极） | **游戏胜利** | 解锁新职业 / 难度模式 |

### 5.3 设计意图

- **想再来一把**：元解锁让"失败局"也有收获，重开有意义
- **故事感**：5 个对手 = 5 章节，对应 stage 1–5/6/7，每个对手是当年的"反派"
- **组合深度**：本局 buff 之间有联动（+收入 + CCR + 反制 BM 可叠）
- **胜利感强烈**：3 张卡即时可用，且和 archetype 故事性绑定——"你打败的对手，他们的人才加入你"

## 6. UI / 视觉规格（遵循 design.md）

### 6.1 血条本体

**形态**：像素风对决血条，红蓝双向挤压式（红=对手，蓝=玩家）。

- **结构**：单一水平长条，中间是"前线"分界。蓝色从左向右增长、红色从右向左增长，交界点 = `playerShare / 100` 的位置。
- **像素硬边**：2–3px 实心深色边框（深蓝黑 `#0a1428` 系），无圆角，左右两端用 `clip-path` 做小斜切角，与卡牌/按钮统一。
- **色彩**：
  - 玩家蓝：`#3aa6ff → #2b6cff` 纵向渐变（信息/协同色，呼应 design.md 蓝=信息）
  - 对手红：`#ff4d4d → #c8202b` 纵向渐变（警示/对抗色）
  - 内部加 1–2px 像素扫描线纹理增强复古感
- **份额数字**：
  - 玩家 % 放血条**左端**内部，蓝色背景 + 近白字，`Fusion Pixel SC`，比 HUD 主数字略小
  - 对手 % 放血条**右端**内部，红色背景 + 近白字
  - 数字带轻微抖动/扫光动效，仅在份额变化时触发

### 6.2 交界处火焰特效

- 复用 `PhaserBattleFX.jsx` 的粒子系统（已有 `'spark'` `'star4'` particle 资源）。
- 在血条 DOM 之上叠一层 absolute 定位的 Phaser canvas（宽 ~60px，高=血条高×1.8，跟随交界点 x 坐标移动）。
- 粒子配置：橙黄火焰色（`#ffaa33` 主、`#ff5522` 辅），向上短距发射 + 左右轻微飘散，life ~600ms，emission rate 与"近 1 月份额变化绝对值"挂钩：变化大→火焰大；静止时只有微弱余烬。
- 玩家份额涨时火焰偏向红方（"烧向敌人"），降时偏向蓝方。

### 6.3 位置与顶部 HUD 重排

血条出现在 `top-hud` 的**正中**，位于 `hud-brand`（左：阶段/融资轮数+日期）与 `hud-actions`（右：combo+设置）之间。

**非对决期**：血条不渲染，HUD 维持现状（`hud-stats-group` 三格大显示）。

**对决期 HUD 缩放规则**：
- 血条占据原 `hud-stats-group` 的水平位置，居中
- 估值 / 现金 / 月度运营成本 三项**文字化**：去掉图标和进度条，只保留"标签 + 数值"形如 `估值 V12,340 ｜ 现金 ¥3,210 ｜ 运营 -¥420`
- 这三项缩成一行小字（font-size 缩到原 70%），放在**血条正下方居中**
- **AP HUD 不变**（保留原位、原大小、原视觉），它是每月主操作必看，对决期反而更重要
- `hud-brand`（阶段/日期）和 `hud-actions`（combo/设置）保持原位原大小

### 6.4 对决感的额外细节

- **血条入场**（对决开始第 1 月）：从中心炸开 → 拉伸到全宽 ~700ms，伴随短促镜头震动（已有 `PhaserBattleFX` 可触发），中央弹字"宿敌登场"+对手名（design.md §8 "重大事件允许中心弹字、镜头轻震"）
- **每月份额变化**：变化数值（如 `+8%`）从交界点向自己方向飘出 → 渐隐 ~1.2s
- **危险线**：玩家份额跌破 30%，血条蓝色侧脉冲红光警告 + 边框闪烁
- **胜利在望**：份额 ≥70% 时蓝色侧出现金色描边（design.md "金=奖励"语义）
- **胜利瞬间**：火焰扩散全屏 → 血条全蓝 + 金色描边定格 → 收购弹窗（见 5.1）
- **退场**（胜/负/超时）：血条收缩回中心爆开 → 顶部 HUD 还原大尺寸

### 6.5 字体与文字

- 血条上方/下方所有数字与标签统一用 `Fusion Pixel SC`
- 中央可在血条上方加一个小标签 `「市场份额对决 · 第 X 月 / 6」`，灰蓝色小字
- 对手名（如"价格屠夫·力恒电商"）显示在红色侧上方小字

## 9. 验证

### 9.1 PR 内自动测试

- `npm test` — 5+ 条新单元测试全过
- `simulation.js` headless 跑 100 局，统计：
  - 每个对手击败率应在 40–70%（tier 1 高、tier 5 低）
  - 平均存活月数 vs 当前版本（应基本持平或略低）
  - 划道费触发率 < 20%（说明大部分输是超时，少数是被压制）
  - 胜利后获得的 3 张卡使下个对手胜率提升约 5–10%（验证奖励有意义但不破坏平衡）

### 9.2 手动验收

1. dev server 起本地，跑一局到第 13 月，确认：
   - 第 9 月看到对手预告，档案完整可见
   - 第 13 月份额血条从中心炸开入场、初始 50/50、火焰在中点燃起
   - 顶部 HUD：估值/现金/运营缩成文字行下移到血条下方，AP HUD 保持原大小
   - 每月正常部署 + 竞争行动可二选
   - 收入差大时血条变化明显（火焰加大、向弱方偏移），收入接近时缓慢
   - 份额跌破 30% 出现警告脉冲、突破 70% 出现金色描边
2. 故意输一次，确认划道费扣款 + 估值降 + 游戏继续 + 血条退场动效
3. 跑通整局到 5 号对手击败 → 收购弹窗显示获得的 3 张卡 → 游戏胜利触发
4. 视觉一致性核查：血条像素边框、斜切角、字体（Fusion Pixel SC）、色彩语义（红=对抗、蓝=玩家、橙=火焰、金=胜利描边）符合 design.md

## 10. 不在本版本范围（后续可加）

- 月度 dilemma（30 秒选择层）
- 半年永久 perk（3 分钟成长层）
- 研发储备等新属性
- 玩家可主动挑战对手（不等到预告）
- 多对手同时存在
