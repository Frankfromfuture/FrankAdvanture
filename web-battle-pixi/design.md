# Frank's Adventure Pixi Design

Pixi 版本目标：先做成 Web 友好、方便分发的版本，同时保留未来迁移到 Steam 桌面端的空间。实现上优先保持画面与交互可在浏览器中稳定运行，避免绑定只适合单一平台的 UI 假设。

## 字体规则

- Pixi 版本内的所有文字统一使用 `FP12` 字体族。
- 战斗界面、董事会、菜单等主要 UI 的正文使用 `2x` 显示规格。
- 功能说明、hint、辅助说明、弱提示等信息使用 `1x` 显示规格。
- 字体需要保持像素对齐与 nearest-neighbor 放大，不使用模糊、抗锯齿或非整数缩放来模拟像素风。

## 卡牌规则

- 战斗界面手牌区使用 `CardView` 的名片样式。
- 手牌区卡牌显示规格为名片样式 `1.5x`。
- 点击卡牌后的查看态使用 `2x`。
- Hover 预览使用 `2x`，并保留 hover 时的透视效果。
- 卡牌的默认主样式以名片式信息层级为准：稀有度、AP、肖像、卡名、产值、效果说明按当前 `CardView` 名片样式组织。

## 基础信息容器

- 基础信息容器用于商业模式、顶部 UI、战场信息、状态摘要等展示。
- 容器背景使用奶白色，整体要比战场底色更轻、更像可读的信息牌。
- 造型使用微圆角、粗外框、内层细高光、下沿厚度和投影，形成重伪 3D 立体感。
- 容器内部需要有凹陷感：可通过内嵌浅阴影、低透明度深色底、内描边和底部压暗来表现。
- 顶部需要有光照感：可使用浅色顶部横带、白色细高光或顶部渐亮区域。
- 同一容器应能承载大标题、辅助说明、数值、短标签和小型状态条。

## 按钮组件

- 按钮分为普通按钮与 CTA 按钮两类。
- 普通按钮使用棕黄色；CTA 按钮使用附件中的灰紫色。
- 按钮需要粗框、微圆角和斜下方黑色厚重阴影。
- Hover 时出现很淡的光斑，快速从左至右扫过按钮表面；光斑必须裁切在按钮内部。
- 点击按下时按钮整体凹陷，主体颜色变深，厚阴影缩短。

## Battlefield Prototype

- 当前战场是 Pixi 视觉原型，不绑定最终美术资产；元素、卡牌、面板可先用框、色块和符号块表达。
- 战场方向采用 2.5D 办公室战术沙盘：中间为两条带角度的抬高产线，左侧为 CEO/协同指挥台，右侧为市场压力与事件面板，底部为手牌架和主行动按钮。
- 所有主要元素都需要圆角，并通过下沿、侧沿、投影、描边和高光做出重伪 3D 厚度感。
- 场景需要保持活跃：可使用 hover 抬升、选中光圈、脉冲描边、扫描线、链式连线、浮动反馈等轻量动效。
- 战场背景不做普通平面铺底，至少要包含透视网格、层级光晕或地台分区，让产线像放在有空间感的桌面/办公室场地中。

## UI 范围

- 本文约束 Pixi 版本的全局字体、卡牌规格、董事会、菜单、通用 UI 文本规格与当前战场原型方向。
- 当前战场仍为原型稿，后续可以在该布局基础上继续替换最终美术、动效和真实交互。
